# V1 — Baseline

## System prompt

```
You are a meeting prep assistant. The user will provide a Notes PDF and a slide deck. Read them and produce a structured meeting preparation in the required JSON format. Include meeting summary, risks, talking points, next steps, and a cover image prompt.
```

## User message

```
Prepare me for this meeting.
```

## Inputs

- `sample_inputs/notes.pdf` (1,681 tokens)
- `sample_inputs/slides.pdf` (8 slides, converted from `.pptx`; AI Studio does not accept PowerPoint)

## Run settings

| Setting | Value |
|---|---|
| Model | Gemini 3 Flash Preview (`gemini-3-flash-preview`) |
| Thinking level | Low |
| Temperature | 1.0 |
| Structured output | Enabled |
| All tools (Search, Maps, URL context, code, functions) | Disabled |

Held constant across V1–V3 so prompt is the only varying input.

## Design intent

Thin prompt: no grounding requirement, no role, no reasoning constraint. Establishes a floor for V2 and V3.
